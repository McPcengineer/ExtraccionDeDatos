# Alura Store – Análisis de Ventas y Rendimiento de Tiendas

## Descripción del proyecto

Este proyecto corresponde al **Desafío Alura Store – Data Science Latam**, cuyo objetivo es realizar un análisis de datos para apoyar la toma de decisiones estratégicas del Sr. Juan, propietario de cuatro tiendas de la cadena Alura Store.

A través del análisis de ventas, calificaciones de clientes y costos operativos, se busca identificar **cuál de las cuatro tiendas presenta el menor desempeño general** y debería considerarse para su venta, permitiendo invertir en un nuevo negocio.

El proyecto simula un escenario real de trabajo para una persona analista de datos, utilizando **Python y Pandas** en un entorno de **Google Colab**.

---

## Objetivo

Evaluar el rendimiento de las cuatro tiendas de Alura Store con base en los siguientes criterios:

- Facturación total por tienda  
- Categorías de productos más vendidas  
- Calificación promedio de los clientes  
- Productos más y menos vendidos  
- Costo promedio de envío  

Con estos análisis, se emite una recomendación final basada en datos para la toma de decisiones estratégicas.

## Estructura del proyecto

El análisis se desarrolla en un notebook de Google Colab con la siguiente estructura:

- **Importación de datos**  
  Carga de las bases de datos de las cuatro tiendas desde GitHub.

- **Análisis de facturación**  
  Evaluación de los ingresos totales generados por cada tienda.

- **Ventas por categoría**  
  Identificación de las categorías de productos más populares en cada tienda.

- **Calificación promedio de la tienda**  
  Análisis de la satisfacción del cliente a partir de las evaluaciones recibidas.

- **Productos más y menos vendidos**  
  Detección de los productos con mayor y menor demanda.

- **Costo promedio de envío**  
  Comparación de los costos logísticos entre las tiendas.

- **Conclusión final**  
  Recomendación sobre cuál tienda debería venderse.

---

## Fuente de datos

Los datos utilizados provienen del repositorio oficial del desafío **Alura Store – Data Science Latam**.  
Cada base de datos corresponde a una tienda distinta y contiene información sobre:

- Producto  
- Categoría  
- Precio  
- Costo de envío  
- Fecha de compra  
- Calificación del cliente  
- Ubicación de compra  

Los datos se cargan directamente desde GitHub y no requieren modificaciones previas.

---

## Metodología de análisis

El análisis se realizó utilizando técnicas básicas de **análisis exploratorio de datos (EDA)**, incluyendo:

- Estadísticas descriptivas  
- Agrupación y agregación de datos  
- Comparación de métricas clave entre tiendas  

Los resultados obtenidos permiten identificar patrones de venta, niveles de satisfacción del cliente y eficiencia operativa.

---

## Resultados y conclusión

A partir del análisis de los cinco criterios definidos, se identificó la tienda con **menor rendimiento general**, considerando aspectos financieros, operativos y de experiencia del cliente.

Esta información proporciona una base sólida para que el Sr. Juan tome una **decisión estratégica informada** sobre la venta de una de sus tiendas e inversión en un nuevo negocio.
